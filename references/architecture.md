# 架构设计文档

> 铁壁四规的技术架构和设计理念

---

## 设计原则

1. **模块化**：四规独立，可单独使用
2. **插件化**：L2层可选可扩展
3. **通用性**：适用于所有Hermes/OpenClaw用户
4. **易用性**：一键安装，零配置使用

## 数据流

### 记忆数据流

```
用户输入
  ↓
Agent处理
  ↓
┌─────────────────────────────────────┐
│           记忆存储判断              │
├─────────────────────────────────────┤
│ 影响行为？ → L1 (MEMORY.md)        │
│ 可检索？   → L2 (Hindsight)        │
│ 原始记录？ → L3 (session_search)   │
└─────────────────────────────────────┘
```

### 代码质量流

```
代码提交
  ↓
┌─────────────────────────────────────┐
│           马诺防线校验              │
├─────────────────────────────────────┤
│ 代码质量关 (ruff/dart analyze)     │
│ 逻辑测试关 (pytest/flutter test)   │
│ 安全扫描关 (pre-commit)            │
└─────────────────────────────────────┘
  ↓
用户验证
  ↓
合并/部署
```

## 扩展点

1. **L2插件**：通过 `plugins/l2-interface.md` 接口扩展
2. **平台适配**：通过安装脚本自动检测
3. **工具链**：通过配置文件自定义
