# 铁壁四规 - 适用场景与限制

> 帮助你判断这个Skill是否适合你的项目

---

## ✅ 适用场景

### 最佳效果场景

| 场景 | 说明 | 效果 |
|------|------|------|
| **独立开发者** | 一个人开发产品 | 四规自动保障质量/文档/记忆/协作 |
| **OPC一人公司** | 一人+AI团队创业 | 铁壁四规是AI协作的基础设施 |
| **多Agent协作** | 2-5个Agent协同工作 | 工蚁规范自动调度，避免混乱 |
| **长期项目** | 持续迭代的产品 | 历史书记住所有决策，不会遗忘 |
| **代码密集型** | 大量代码编写 | 马诺防线保证代码质量 |
| **文档要求高** | 需要规范文档 | 图书馆自动归档管理 |

### 典型项目类型

- SaaS产品开发
- 移动App开发（Flutter/RN）
- Web应用开发
- AI工具/Agent开发
- 技术博客/内容平台
- 开源项目维护

## ⚠️ 使用限制

### 硬性限制

| 限制 | 说明 | 解决方案 |
|------|------|----------|
| **不支持Windows** | 仅支持Linux/macOS | 使用WSL2或Docker |
| **需要bash环境** | 安装脚本依赖bash | macOS/Linux自带 |
| **L2层需Python** | Hindsight插件需要Python 3.10+ | 安装Python或选Null插件 |
| **单平台运行** | 不能同时在Hermes和OpenClaw运行 | 选择一个平台 |

### 软性限制

| 限制 | 说明 | 建议 |
|------|------|------|
| **小型项目** | 1-2天能完成的项目 | 四规可能过重，手动管理即可 |
| **一次性脚本** | 不需要迭代的脚本 | 不需要文档管理和记忆系统 |
| **纯运维任务** | 只部署不开发 | 马诺防线用不上 |
| **单人无Agent** | 一个人不用Agent | 工蚁用不上，其他三规可用 |

### 已知问题

| 问题 | 状态 | 说明 |
|------|------|------|
| Hindsight国内安装 | ⚠️ 需配置镜像 | 设置 `HF_ENDPOINT=https://hf-mirror.com` |
| 大文件索引慢 | 📝 设计如此 | L2层不适合存储大文件，只存知识摘要 |
| 首次加载慢 | 📝 正常 | Hindsight首次需加载模型，约10-30秒 |

## 🎯 快速判断

**适合你吗？回答以下问题：**

1. 你是否在用AI Agent（Hermes/OpenClaw）？
   - 是 → 继续
   - 否 → 不需要

2. 你的项目是否需要迭代（不是一次性的）？
   - 是 → 继续
   - 否 → 可能不需要

3. 你是否需要代码质量管理？
   - 是 → 马诺防线适合你
   - 否 → 其他三规仍可用

4. 你是否有2个以上Agent协作？
   - 是 → 工蚁非常适合
   - 否 → 工蚁暂时用不上

**结论：**
- 4个都是"是" → 强烈推荐
- 3个是"是" → 推荐使用
- 2个是"是" → 可以使用
- 1个或0个是"是" → 不需要
