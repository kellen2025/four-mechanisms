# TRACE 评测体系

> SkillHub.cn 官方技能质量评测标准（5.0满分体系）

来源：https://skillhub.cn/tutorials#trace-evaluation

## 五个维度（由基础到价值）

### T - Trust 可信任度
- 最小权限原则（不要求无关高权限）
- 输入验证（防止注入攻击）
- 敏感数据保护（不暴露用户隐私）
- 依赖安全（可溯源、无恶意代码）
- 国内网络可直接跑通
- 完整支持中文交互

### R - Reliability 可靠性
- 同样任务每次正常执行
- 异常输入合理处理
- 失败时给出可理解反馈
- 常见/边界/复杂输入下保持一致表现

### A - Adaptability 适用性
- Skill与用户需求匹配精度高
- 能力边界清晰
- 语义匹配准确（不误触发、不漏调用）

### C - Convention 规范性
- 可读性好（快速理解能解决什么问题）
- 可维护性好（快速定位如何扩展/修复）
- 渐进式信息披露
- 可复用（从一次性能力沉淀为可复用资产）

### E - Effectiveness 有效性
- 结果正确
- 输出完整
- 符合预期格式
- 可直接使用
- 提供额外价值（信息整合、判断建议）

## 通用Skill发布规范

### SKILL.md 元数据必须包含
```yaml
---
name: skill-name
version: X.Y.Z
description: "清晰的功能描述"
category: 分类
author: 作者
license: 协议
platforms: [hermes, openclaw]
tags: [标签1, 标签2]
---
```

### 安全审核要求（腾讯标准）
1. 无恶意代码（无rm -rf、无数据窃取）
2. 无隐私泄露（不收集用户数据）
3. 依赖透明（所有依赖可溯源）
4. 文档完整（安装/使用/卸载说明齐全）
5. 版本清晰（语义版本号+变更日志）

### 版本号规范（语义版本号）
- 主版本（Major）：不兼容的API变更
- 次版本（Minor）：新增功能（向下兼容）
- 修订号（Patch）：问题修复（向下兼容）
