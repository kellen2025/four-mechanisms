# 每日三层记忆整理工作流

> Job ID: 1cda8dbdd7f0 | 执行时间: 每天20:00 | 交付: 本地

## 触发条件

每天20:00自动执行，或用户手动要求整理当天记忆。

## 执行步骤

### 1. 扫描当天会话

```
session_search(query="", sort="newest", limit=10)
```

筛选出当天日期的会话记录。

### 2. 信息分类与存储

对每个会话提取有价值信息，按三层规则分类：

#### L2层 - Hindsight（温存储）— 主要目标

以下信息**必须**retain到Hindsight：
- 用户纠正/反馈（"不要这样做"、"应该这样做"）
- 技术决策和理由（"选择X因为Y"）
- 项目里程碑和进展
- 新发现的工具/API/技巧
- 踩坑记录和解决方案
- 重要的架构/设计变更

#### L1层 - MEMORY.md（热缓存）— 仅当影响行为

以下信息如果影响Agent行为，更新MEMORY.md：
- 新增的用户偏好
- 新增的铁律/规则
- Agent团队变更
- 关键环境事实变更

#### L3层 - session_search（冷存储）— 无需操作

原始会话已自动存储，无需手动操作。

### 3. 执行retain

对每条需要存储的信息，调用hindsight_retain()：
- content: 信息内容（简洁完整）
- context: 上下文说明
- tags: 标签分类

### 4. 输出整理报告

```
=== 每日记忆整理报告 - YYYY-MM-DD ===
扫描会话数：X
新增Hindsight记忆：X条
更新MEMORY.md：是/否
关键发现：...
```

## 重要规则

- 只存储有价值的信息，不存储临时/琐碎内容
- 信息要简洁完整，便于未来recall
- 标签要准确，便于检索
- **不重复存储已有信息**（先recall检查）
- 遵循写入验证流程（来源检查→矛盾检查→可发现性→写入后校对）

## 与L1压缩的关系

- L1压缩时迁移的内容 → 通过retain自动进入L2
- 每日整理 → 补充L1未覆盖但L2需要的内容
- 两者互补，不冲突
