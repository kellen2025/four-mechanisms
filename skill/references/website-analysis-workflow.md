# 系统化网站抓取分析工作流

## 适用场景

需要深度分析一个网站的结构、功能、商业模式、技术架构时使用。常见于竞争对手分析、行业研究、产品参考调研。

## 工作流

### 阶段一：结构探索（米宝执行）

使用浏览器工具快速扫描网站主页和核心子页面，建立整体认知：

1. **浏览首页** — 确认网站名称、备案号、运营主体、定位口号
2. **浏览开奖/核心数据页** — 每个彩种的开奖结果页面结构
3. **浏览工具/功能页** — 走势图、工具箱、筛选器等
4. **浏览404/地图页** — 404页面通常暴露完整子页面导航结构
5. **记录关键URL和功能模块** — 为子Agent提供抓取目标

### 阶段二：并行深挖（子Agent执行）

根据第一阶段的结构认知，将抓取任务分解为3个并行子任务：

#### 任务A：预测推荐体系
- 抓取所有预测页面（专家排行、达人秀、杀号定胆等）
- 汇总预测推荐类型
- 分析排名机制

#### 任务B：走势图和技术工具
- 抓取走势图页面（新旧版均需）
- 汇总所有技术工具
- 收集技巧文章列表

#### 任务C：商业分析和整体架构
- 分析商业模式（虚拟货币、付费层级）
- 分析竞争定位
- 评估技术架构

**每个子Agent的goal应明确、scope限定，context包含第一阶段已发现的URL和结构信息。**

### 阶段三：报告编译（米宝执行）

汇总三个子Agent的产出，编译为统一格式的报告：

```
## 一、网站概览           ← 基础信息
## 二、网站架构与导航       ← 页面结构
## 三、核心功能模块分析      ← 功能详述
## 四、XXX专题分析         ← 各子Agent深挖产出
## 五、商业模式分析         ← 收入/货币体系
## 六、用户画像/竞争/技术评估 ← 综合分析
## 七、总结
```

### 阶段四：归档到图书馆

1. 创建目录：`~/.hermes/tushuguan/website-analysis/<site-name>/`
2. 写入报告：`YYYY-MM-DD_<site-name>-analysis-report.md`
3. 更新 INDEX.md：
   - 在目录结构部分确认包含 `website-analysis/`
   - 在[网站分析]表格中新增一行
   - 在变更日志中记录新增

## 命名规范

| 项目 | 规范 |
|------|------|
| 目录 | `website-analysis/<site-name>/` (全小写，连字符) |
| 文件名 | `YYYY-MM-DD_<site-name>-analysis-report.md` |

## Pitfalls

### 误区：不先探索结构就派子Agent
**问题**：直接让子Agent爬全站，但没给它们任何URL和页面结构信息。
**正确做法**：米宝先浏览首页和核心页面，记录关键URL和模块结构，再派子Agent。

### 误区：子Agent的goal太宽泛
**错误**："分析整个网站"
**正确**："分析双色球预测页面，包括专家排行、达人秀、杀号定胆三大板块"

### 误区：子Agent并行抓取相同页面
**正确做法**：每个子Agent的goal互斥，覆盖不同板块。相同URL只由一个Agent爬取。

### 误区：直接拼贴子Agent输出作为报告
**正确做法**：米宝编译成统一格式，去重、整合、补充gap，形成连贯的分析报告。
