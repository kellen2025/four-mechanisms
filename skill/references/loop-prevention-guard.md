# 防循环陷阱操作手册

> 2026-06-16/17 会话中多次发生无限循环，总结为操作规范

## 陷阱1：bash `*` 通配符导致语法错误循环

**场景**：设置环境变量时，`API_KEY=*** 中的 `*` 被bash解释为通配符。

**错误命令**：
```bash
export API_KEY=*** ~/.config/ima/api_key)  # * 被展开为文件名
```

**正确写法**：
```bash
# 方法1：用 $() 读取文件内容
API_KEY=*** ~/.config/ima/api_key)

# 方法2：用单引号保护
API_KEY='***'  # 但这不是从文件读取
```

**规则**：bash中 `*`、`$`、`` ` ``、`(`、`)`、`{`、`}` 都有特殊含义，必须用 `$()`、单引号、或转义来保护。

## 陷阱2：read_file 被BLOCKED后重复调用

**场景**：read_file 返回 "BLOCKED: You have called read_file on this exact region N times"，但Agent继续调用相同参数。

**2026-06-17 实测**：Agent连续调用 read_file 相同区域 **146次**，直到系统强制阻止。

**根本原因**：Agent没有检查返回结果中的BLOCKED警告，机械重复相同操作。

**正确行为**：
```
收到 BLOCKED 警告 → 立即停止 → 换用其他工具（search_files、terminal）
```

**规则**：
- read_file 返回 BLOCKED → 换方法，不再调用
- terminal 命令返回非0 → 分析stderr，不盲目重试
- 同一工具连续失败3次 → 停止 → 报告用户

## 陷阱3：terminal 命令语法错误循环

**场景**：terminal 命令因语法错误失败，Agent连续重试相同命令30+次。

**2026-06-17 实测**：bash语法错误 `未预期的记号 ) 附近有语法错误`，Agent重试 **33次**相同命令。

**根本原因**：Agent没有分析错误信息，假设"重试可能成功"。

**正确行为**：
```
命令失败 → 读取stderr → 分析错误类型 → 修正命令 → 重试1次
                    ↓
              如果仍失败 → 报告用户，请求指导
```

## 自动跳出检查清单

每次工具调用失败后：
1. ✅ 检查返回结果是否包含 BLOCKED/Error/failed
2. ✅ 如果是，分析错误原因（不要假设重试会成功）
3. ✅ 连续失败3次 → 停止 → 报告用户
4. ✅ 换用不同工具或方法
