# 铁壁三：历史书 (History Book)

> 三层记忆架构，让知识按热度分层存储

---

## 架构总览

```
┌─────────────────────────────────────────────────────┐
│                    历史书                            │
├─────────────────┬─────────────────┬─────────────────┤
│    L1 热缓存    │    L2 温存储    │    L3 冷存储    │
│   MEMORY.md     │  可插拔工具层   │ session_search  │
│  每轮自动注入   │   按需检索      │  全自动存储     │
│    零延迟       │    ~200ms       │   按需检索      │
└─────────────────┴─────────────────┴─────────────────┘
```

---

## L1 热缓存 (MEMORY.md)

**存储内容**：影响Agent行为的规则/偏好

**触发条件**：
- 新增铁律/规则
- 新增用户偏好
- Agent团队变更
- 关键环境事实变更

**限值**：
- 米宝 (default): 4000 字节
- 子Agent: 2000 字节

**防护机制**：
- 80%预警：触发自动压缩
- 100%硬限制：拒绝写入

---

## L2 温存储 (可插拔工具层)

**存储内容**：可检索的历史知识

**触发条件**：
- 用户纠正/反馈
- 技术决策和理由
- 项目里程碑和进展
- 踩坑记录和解决方案

**插件选择**：
| 插件 | 说明 | 推荐度 |
|------|------|--------|
| Hindsight | 功能完整 | ⭐⭐⭐⭐⭐ |
| Null | 不使用L2 | ⭐⭐ |
| 自定义 | 按需扩展 | - |

详见 `plugins/l2-interface.md`

---

## L3 冷存储 (session_search)

**存储内容**：原始会话记录

**触发条件**：全自动，无需操作

**检索方式**：session_search(query="关键词")

---

## 写入验证流程

```
Agent准备写入
  ├─ ① 来源检查：用户告知 / 工具验证 / 自己推断(降级)
  ├─ ② 矛盾检查：与已有记忆矛盾 → 停止，问用户
  ├─ ③ 可发现性：一条命令能查到 → 不存
  └─ ④ 写入后校对：重新读取确认
```

---

## 准入标准

**核心判断问题**：这条信息如果这次会话没有注入系统提示，会不会导致我犯错？

- **会犯错** → 存 MEMORY.md（L1）
- **不会犯错** → 存 Hindsight（L2）或不存

| 类别 | L1 | L2 | 不存 |
|------|----|----|------|
| 铁律规则、用户偏好、关键环境事实 | ✅ | | |
| 项目进展、技术经验、实体关系 | | ✅ | |
| 临时任务进度、可重新发现的信息 | | | ✅ |
