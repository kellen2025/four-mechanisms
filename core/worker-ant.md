## 任务分配

### 混合调度模式

```
用户需求
  ↓
① 主控：判断任务性质
   ├─ 简单（≤3步/单Agent/无需评审）→ 直接执行
   └─ 复杂（多步骤/跨Agent/需评审）→ 任务卡片
  ↓
② 复杂流程（自动门控）：
   产品 → 算法 → 编码 → 运营 → 验收
  ↓
③ 调度器每60s调度ready任务
   Worker按任务体执行 → 完成 → 自动升ready
  ↓
④ 主控验证产出 → 构建产物 → 交付用户
```

### 调度原则

- 简单任务：单Agent直接执行
- 复杂任务：多Agent协作
- 专业任务：匹配专业Agent

## 进度追踪

### 状态流转

```
pending → ready → processing → completed
                    ↓
                  failed → retry → processing
```

### 状态同步

- 每个任务有唯一ID
- 状态变更实时同步
- 失败任务自动重试（最多3次）

---

## 超时应对

### 三级升级机制

| 级别 | 条件 | 动作 |
|------|------|------|
| L1 | 正常时间内完成 | 正常接收 |
| L2 | L1超时 | 预读关键数据 → 传给新Agent |
| L3 | L2超时 | 诊断日志 → 修复 → 重调度 |

### 配置参数

```yaml
delegation:
  child_timeout_seconds: 1800
  max_iterations: 50
  max_concurrent_children: 3
  max_spawn_depth: 1
```
